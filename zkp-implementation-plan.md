# Sentinel ZKP Integration Implementation Plan

## Goal Summary
Upgrade the existing ECDSA-based compliance firewall to a true Zero-Knowledge Proof (ZKP) system using **Circom** and **SnarkJS**, as outlined in Phase 2 of `walkthru.md`. The Sentinel API will locally generate a Groth16 snark proof confirming a user's balance without revealing the actual balance on-chain, and the destination smart contract will verify it natively.

## User Constraints & Requirements (What You Need to Provide)
Before we start, I need you to confirm or provide the following:
1. **Circom Compiler Installation**: Since you are on Windows, you will need to have the `circom` compiler installed globally (usually via Rust/Cargo or pre-built binaries). Please ensure `circom` is accessible in your command line, or let me know if you need help installing it.
2. **Trusted Setup (Powers of Tau)**: To compile the circuit and generate the `.zkey`, we need a Powers of Tau (`.ptau`) file. I can write a script to automatically download a public `pot12_final.ptau` from Hermez network. Let me know if that is acceptable.
3. **Mock Bank Data**: Currently, `mock-bank` just returns `{ qualified: true }`. To make the ZKP realistic, I propose updating it to return `{ balance: 200000 }` so the Sentinel API can dynamically fetch this for the `userBankBalance` signal. Should I make this change?

## Proposed Changes

### 1. ZKP Circuit Development
- **[NEW]** `dynamic-compliance-firewall/circuits/compliance.circom`: Create the circuit to prove `userBankBalance >= requiredBalance` using `circomlib` comparators.
- **[NEW]** `dynamic-compliance-firewall/scripts/compile-circuit.sh`: A shell script to compile the circuit, download the `.ptau` file, run the Groth16 setup, and generation of `Verifier.sol`.
